# Markdown 视图简化与正确性修复

## 目标

- Rendered 与 Raw 保持在同一个 Markdown tab 内切换，大纲不因正文模式变化而卸载、折叠或丢失宽度。
- 大纲点击、滚动定位和活动标题使用同一条视口激活线，不再高亮到上一个标题。
- 跨视图恢复只以源码行为稳定基准，删除 block、滚动比例和多帧重试组成的通用锚点状态机。
- 长文档继续渐进挂载，但只按真正的大文件触发，并将分块、调度和目标块提升压缩为最小实现。
- 保留离屏代码块延迟高亮和大纲虚拟滚动。
- 拖动大纲分隔线时显示真实、清晰的竖向预览线，释放指针后才提交正文宽度。
- 删除绑定旧实现的测试，以少量行为测试覆盖用户可见的不变量。

## 明确取舍

- 不增加渐进渲染开关或 Settings 项。
- 不再把 Markdown tab 转换成 Editor tab。
- Raw 编辑仍复用完整 `EditorPane`，保留保存、搜索、关闭保护、外部文件刷新和编辑器快捷键。
- 跨视图恢复定位到当前源码行，不保证段落内部完全相同的像素位置。
- 大纲虚拟滚动继续保留，不在本次重写。
- 渐进渲染只为超大文档承担复杂度。普通文档始终使用 Streamdown 静态模式。

## 实施步骤

### 1. 稳定 Markdown shell

- 让 `MarkdownTab` 直接持有 `dirty`，删除 tab kind 转换和 `setMarkdownView`。
- `viewMode` 只由已挂载的 Markdown pane 持有，不进入持久化 tab 模型。
- `MarkdownStack` 在同一分栏正文区域内切换 Rendered 与懒加载的 `EditorPane`。
- 将 Markdown Raw 接入编辑器 handle、dirty 标记、关闭保护、搜索、快捷键和文件监听。
- 文档模型与大纲由外层 shell 持有，切换 Raw 时大纲继续挂载。

### 2. 源码行恢复与大纲定位

- 将跨视图状态缩减为单个源码行。
- Rendered 通过轻量 source-position 属性读取和恢复源码行。
- Raw 通过 CodeMirror 顶部可见行读取和恢复源码行。
- 大纲活动标题优先根据统一激活线上的源码行计算，缺少源码位置时再回退到标题 DOM。
- 点击标题先确保目标块挂载，再滚动到同一激活线并立即更新活动项。
- 修复 fenced code 的 source-position 属性落在被替换节点后丢失的问题。

### 3. 渐进渲染精简

- 删除按 block 数量触发的阈值，避免短小碎片文档进入渐进模式。
- 使用更高的文件大小阈值，让普通文档走静态渲染。
- 只计算渲染需要的块字段，删除 hash key、lineCount 等无消费者数据。
- 用 `Uint8Array` 和按块通知的小型 store 替换防御性过强的 controller。
- 首屏挂载、空闲批量挂载和大纲目标块提升保留；无 `requestIdleCallback` 时按批处理，避免逐块延迟数十秒。
- 对跨块引用、脚注等不能安全独立渲染的文档回退静态模式，正确性优先。

### 4. 分栏拖动

- 保留指针释放后一次性提交宽度的策略。
- 使用真实绝对定位元素显示拖动预览线，不再依赖 separator 伪元素和 CSS 变量。
- 合并宽度状态和拖动状态，删除重复的 rendered、preferred 和 ghost 同步逻辑。
- 保留键盘调整、窄窗口自动折叠和可访问属性。

### 5. 测试精简与验证

- 删除读取源码字符串判断组件结构的测试。
- 删除旧 controller、block key、block 数阈值和通用锚点实现细节测试。
- 保留 Markdown 语义边界、标题提取和安全回退测试。
- 用单元测试覆盖统一标题激活、源码行恢复、远距离块提升和批量调度。
- 在真实 Tauri WebView 中验证 Rendered/Raw 保持大纲、点击定位、长文档挂载和拖动预览线。

## 验收标准

- Rendered 与 Raw 往返时，大纲保持展开状态和已提交宽度。
- Raw 编辑仍能保存，dirty 标记和关闭确认正常。
- 点击任意大纲标题后，正文定位与活动高亮一致，不停留在上一个标题。
- 代码块附近的活动标题和跨视图源码行定位正确。
- 普通文档不进入渐进渲染；超大文档首屏可用，远距离大纲跳转能立即挂载目标。
- 无 `requestIdleCallback` 的 WebView 不再每 16ms 只挂载一个块。
- 拖动期间正文不连续重排，竖向预览线清晰跟随指针。
- 新增生产代码明显少于被替换实现，不保留兼容开关和无消费者字段。
